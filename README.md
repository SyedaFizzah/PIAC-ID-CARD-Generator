# PIA Intern ID Card System

## What's here
- `backend/` — FastAPI + Postgres + Pillow card generator
- `frontend/` — React + Vite + Tailwind admin dashboard

Phase 1 (this drop): admin login, add intern, auto-generate unique ID, generate + download front ID card.
Phase 2 (later): barcode on back card + attendance scan in/out.

---

## 1. Backend setup

```bash
cd backend
python -m venv venv
venv\Scripts\activate          # Windows
pip install -r requirements.txt
```

Create the database (in psql or pgAdmin):
```sql
CREATE DATABASE pia_intern_db;
```

Copy `.env.example` to `.env` and fill in your Postgres password + a random secret key:
```
DATABASE_URL=postgresql://postgres:YOUR_PASSWORD@localhost:5432/pia_intern_db
SECRET_KEY=some-long-random-string
```

Create your admin login (one-time):
```bash
python create_admin.py
```

Run the server:
```bash
uvicorn app.main:app --reload
```
Backend runs at `http://localhost:8000`. Swagger docs at `http://localhost:8000/docs`.

---

## 2. Frontend setup

```bash
cd frontend
npm install
npm run dev
```
Runs at `http://localhost:5173`.

---

## 3. Using it
1. Go to `localhost:5173`, log in with the admin account you created.
2. Click **+ Add Intern**, fill in name/department/CNIC, upload a photo.
3. On the dashboard, click **Generate Card** → then **Download** to get the PNG.

## Notes on the design
- The card is generated by loading your actual Canva export
  (`backend/app/ID Card Template.png`) as the base image, then pasting the intern's
  photo and drawing their name on top — nothing is hand-drawn/recreated anymore.
  This guarantees it matches the approved design exactly.
- If you move or re-export the template, update `FRONT_TEMPLATE_PATH` at the top of
  `card_generator.py`, or just keep overwriting the same filename in the same spot.
- Layout coordinates (`PHOTO_BOX`, `NAME_POS`, `HEADING_POS` in `card_generator.py`)
  were measured against your 591x1004 template's clear safe zone (roughly y=250 to
  y=630, avoiding the logo and corner accents). If you tweak the template's border
  size, re-check these against the new image dimensions.
- Currently the card only shows: logo (baked into template), photo, name, and
  "INTERN" heading — per your note to drop ID number/department from the visible card.
- The unique ID (`PIA0726001` format, monthly-resetting sequence) IS still generated
  and stored — it's just not printed on the front. It'll be what gets encoded into
  the barcode on the back card in phase 2.
- Drop custom fonts (e.g. matching your Canva serif) into `backend/app/assets/fonts/`
  and reference the filename in `card_generator.py`'s `_font()` calls — right now it
  falls back to DejaVu Sans if no custom font is found.
